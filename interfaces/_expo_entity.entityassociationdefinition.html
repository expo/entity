remarks

The entity framework doesn't prescribe a one-size-fits-all solution for referential integrity; instead it exposes mechanisms that supports both database foreign key constraints and implicit entity-specified foreign keys. Choosing which approach to use often depends on application requirements, and sometimes even a mix-and-match is the right choice.

  • If referential integrity is critical to your application, database foreign key constraints combined with {@link EntityEdgeDeletionBehavior.CASACDE_DELETE_INVALIDATE_CACHE} are recommended.
  • If the database being used doesn't support foreign keys, then using the entity framework for referential integrity is recommended.
