Class EntityMutationTrigger<TFields, TID, TViewerContext, TEntity, TSelectedFields>

A trigger is a way to specify entity mutation operation side-effects that run within the same transaction as the mutation itself. The one exception is afterCommit, which will run within the transaction if a transaction is supplied.
